Splash Screen

Technical Description

The Splash Screen was implemented using a combination of HTML, CSS, and JavaScript to create an engaging intro to the website. The structure was built using semantic HTML elements such as ("div", "img", and "p") to organise content clearly. CSS Flexbox was used in the "body" to centre the main card both vertically and horizontally using ("display: flex", "justify-content: center", and "align-items: center"). A loading animation was implemented using CSS keyframes. The progress bar uses a ".fill" div with an animation that gradually increases width from 0% to 100% over 4 seconds. This visually represents loading progress. The page automatically redirects to the home page that has been in our lecture 3.

Accessibility

Accessibility was considered by including descriptive "alt" text for all images, ensuring that screen readers can interpret visual content. High contrast colours (dark background with light text) were used to improve readability. Clear and simple fonts were used to ensure text is easy to read for all users. The “Skip Intro” button improves accessibility by allowing users to skip the animation. The layout is responsive.

Action Impact Simulator

Technical Description

The Action Impact Simulator was developed using HTML, CSS, and JavaScript to create an interactive user experience. The page structure uses semantic elements such as ("header", "main", and "footer") to improve organisation and readability.CSS Grid was used to create a responsive layout for the action cards using "grid-template-columns". Each action is displayed as a card with images, headings, and descriptions. Hover effects and transitions were implemented to enhance interactivity.JavaScript was used to handle user interaction. When a user clicks a card, the "updateImpact()" function updates the total score.Conditional logic was used to display different feedback messages based on the score, improving user engagement. A reset button was also implemented to allow users to restart the simulation.

Accessibility

Accessibility was supported by providing descriptive "alt" attributes for all images. The interface uses clear headings and readable font sizes to ensure content is easy to understand. The layout is responsive, adapting to smaller screens using media queries. Interactive elements such as cards and buttons are visually highlighted using hover effects, helping users identify clickable elements easily.Simple language and clear instructions were used to ensure the simulator is understandable for a wide range of users.

Content Page

Technical Description

The Content Page was designed using structured HTML and modern CSS techniques to present information clearly. The page uses semantic elements such as "(article", "section", "h1", and "p") to organise content logically. A central container (".article-container") was used to hold the content, styled with a semi-transparent background and "backdrop-filter" to create a glassmorphism effect. This improves visual appeal while maintaining readability. CSS properties such as ("line-height","margin", and "font-size") were used to enhance text readability. Images were styled using "object-fit: cover" and responsive width settings to ensureproper scaling across devices. The page includes headings, paragraphs, and lists to structure information effectively, making it easy for users to scan and understand.

Accessibility

Accessibility was addressed by using clear headings and structured content, making it easier for screen readers to navigate. All images include descriptive "alt" text to support visually impaired users. High contrast colours and readable font sizes were used to ensure content is visible and accessible. The layout is responsive and adapts to different screen sizes, ensuring usability across devices. Simple language and logical content flow improve understanding for all users.

Challenges and Lessons Learned

One challenge was aligning elements using CSS, especially for responsive layouts. This was solved by using "justify-content", "align-items", and media queries .Another challenge was adding JavaScript to update the score in the Impact Simulator. This was improved by learning DOM manipulation and event handling. Maintaining consistent design across pages was also difficult, but it helped improve my UI design skills.Overall, this project improved my understanding of responsive design and JavaScript.

Compliance

The web pages follow proper HTML structure and use semantic elements. Accessibility was improved by adding alternative text for images, using readable fonts, and maintaining good colour contrast. The content is educational and focused on Sustainable Development Goal 14 (Life Below Water). External resources and images were properly referenced.
