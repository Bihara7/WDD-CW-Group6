Home page

Technical Description

The Home Page is the entry point for the 'Blue Horizon' project. The idea behind this page is to engage users with the urgency of UN Sustainable Development Goal 14, 'Life Below Water,' through immersive visual storytelling and data.

Accessibility

The page uses a modular design approach, combining high-impact hero elements with informative card grids to organize complex environmental data.

Structural Logic (HTML)

  • Hero Section : This section uses a containerized approach to overlay a high-definition video background ('.videH') with a centered text card. This ensures the key messege is legible regardless of the video content.
  • Static Box ('.stat-box'): Utilizes a horizontal layout to present critical data points (71%, 8M tons, etc.) in a scannable format. This helps the users digest the scale of the problem quikly before moving to detailed content
  • Card-Based Content ('.cards' and '.explore-grid'): These sections use repeating 'div' structures to present "Why oceans matter" and specific sub-topics. This modularity makes the code easy to maintain and scale.
  • Interactive Buttons: Instead of standard text links, 'nput type="button"' is used for primary calls to action, such as "Learn More," to provide a more app-like interactive feel.

Styling and Design (CSS)

  • Backdrop Filtering: The '.card2' and '.explore-card' classes utilize 'backdrop-filter: blur(5px)'. This modern CSS property allows the background to remain visible while blurring the area directly behind the card, significantly improving text contrast.
  • Visual Hierarchy: Differentiation between '.main-topic-title' and '.sub-topic-title' (using size and case transformations) helps guide the user's eye through the narrative of the page.
  • Transitions: The ':hover' effects on '.card2' (box-shadow and opacity changes) and '.learn-more-btn' (color inversion) provide visual feedback that improves the user's sense of interaction.
  • Fixed Backgrounds: The 'body::before' pseudo-element with 'background-attachment: fixed' allows the background image to stay in place while content scrolls over it, adding a sense of physical depth to the website.

Gallery Page

Technical Description

The Ocean Awareness Gallery is designed to be an interactive visually immersive experience that aligns with the "Life Below Water" theme. The implementation relies on a "Card-to-Modal" workflow where condensed information expands into a detailed view upon user interaction.

Structural Logic (HTML)

  • header and nav: These provide consistent site-wide orientation. The active class on the Gallery link serves as a visual "You Are Here" marker improving user flow.
  • section class="cards": Acts as a flex-container for the gallery items. Each gallery item is contained within a 'div' using custom data attributes (e.g., data-title, data-desc). This logic allows JavaScript to treat the HTML as a mini-database pulling content into the modal dynamically without needing separate pages for each image.
  • 'video' with autoplay loop muted: This creates a dynamic background. Using the muted attribute is a functional necessity; most modern browsers block autoplaying videos that have active audio to prevent a poor user experience.
  • div id="galleryModal": This is a hidden structural element. It remains out of sight until triggered at which point it acts as an overlay to focus the user’s attention on a single topic.

Styling and Design (CSS)

  • display: flex and flex-wrap: wrap: These properties ensure the gallery is responsive. As the viewport narrows, cards automatically wrap to the next line, maintaining a clean grid on both mobile and desktop.
  • background: linear-gradient: The transition from deep navy to black mimics the "Twilight Zone" of the ocean, reinforcing the project’s environmental theme through color psychology.
  • position: fixed and z-index: -1: These are used for the background video. Fixing the position ensures the video stays put while the user scrolls, while a negative z-index ensures it stays behind all text and interactive elements.
  • backdrop-filter: blur(5px): Applied to the cards and modal this creates a "glassmorphism" effect. It improves readability by softening the background video's movement behind the text.

Accessibility

Accessibility was a primary consideration to ensure the gallery is usable for all visitors including those using assistive technologies:

  • Alt Text Strategy: Every 'img' tag includes descriptive alt attributes. This ensures that screen reader users receive the same context regarding marine life (e.g., "Photo of yellow fishes near corals") as sighted users.
  • Interactive Feedback: While the scale transformation (scale(1.05)) is a visual cue, the inclusion of a cursor: pointer property provides a functional cue that the cards are interactive elements.
  • Color Contrast: The use of light blue and white text against a dark desaturated background ensures high contrast meeting readability standards for users with visual impairments.
  • Modal Controls: The modal includes a prominent "Close" button (×), and a JavaScript listener allows users to click outside the content area to exit, providing multiple intuitive ways to navigate back to the main gallery.
  • Customization Tools: The "Change Font" and "Change Color" buttons in the modal allow users to adjust the reading environment to their personal needs (e.g., switching to a monospaced font or a high-contrast dark theme).

Content Page

Technical Description

The Marine Pollution Content Page is designed to deliver dense educational information in a readable high-impact format. It utilizes a "layered" visual approach to separate the core educational content from the atmospheric background.

HTML Structure and Logic

  • article: This semantic tag is used for the .article-container because the content is a self-contained independent report on marine pollution. This helps search engines and screen readers identify the primary focus of the page.
  • section class="content-hero": This creates a focused introduction. By grouping the 'h1' a summary paragraph and the primary "hero" image it establishes the context immediately before diving into technical details.
  • strong and ul: These are used strategically to break up walls of text. 'strong' highlights critical statistics (e.g., 8 million metric tons), while 'ul' categorizes different pollution types making the information "scannable."

CSS Styling and Design Choices

  • rgba(0, 0, 0, 0.7) and backdrop-filter: blur(10px): These properties create the semi-transparent "frosted glass" container. The transparency keeps the user connected to the ocean-themed background while the blur and dark tint ensure the white text remains legible.
  • background-attachment: fixed : Applied to the body this keeps the background gradient stationary while the content scrolls over it creating a sense of depth and immersion.
  • linear-gradient HR: Instead of a standard line the 'hr' uses a transparent-to-skyblue gradient. This acts as a soft visual "breather" between heavy sections without creating a harsh border.
  • object-fit: cover : Applied to the .mid-img this ensures that the hero image fills its container without being distorted or stretched regardless of the screen size.
  • transition: 0.3s and box-shadow : These are used on the .back-btn to provide smooth interactive feedback. The "glow" effect on hover mirrors the bioluminescent or shimmering themes of the ocean.

Accessibility

Specific measures were taken to ensure this information is accessible to all users:

  • Semantic Hierarchy: The page uses a logical heading structure ('h1' for the title, followed by 'h2' for sub-sections). This allows screen reader users to skip directly to specific topics like "Human Health Consequences."
  • Legibility and Contrast: White text on a 70% opaque black background provides a high contrast ratio meeting WCAG standards for readability. The line height is set to 1.8 to prevent "text crowding" which assists users with dyslexia or visual impairments.
  • Descriptive Alt Text: Images include meaningful descriptions (e.g., "Plastic Waste near body of water") rather than generic filenames providing context for users who cannot see the images.
  • Non-Visual Navigation: The "Back to Home" button uses a clear descriptive label rather than a vague "Go Back" ensuring users know exactly where the link will take them.

Challenges and Lessons Learned

Challenges Faced

1. Synchronizing Dynamic Modal Content

The Challenge: Initially the Gallery page required hard-coding a separate modal for every single image card. This led to "code bloat" making the HTML difficult to manage and increasing the risk of broken links if a description changed.

The Resolution: I shifted to a Data-Driven Logic using JavaScript. By storing the title description, and image path within HTML5 data attributes (e.g., data-title, data-desc) on the card itself, I wrote a single, reusable JavaScript function. This function dynamically "injects" the correct content into one master modal template whenever a card is clicked.

2. Ensuring Readability Over Dynamic Backgrounds

The Challenge: The background video and deep-sea gradients while visually striking, made the white text difficult to read in certain sections. Standard black text was invisible and pure white text often "vibrated" against the moving video pixels.

The Resolution: I implemented Glassmorphism techniques. By using rgba(0, 0, 0, 0.7) for container backgrounds combined with backdrop-filter: blur(10px) I created a "shield" for the text. This allowed the background colors to peak through while providing a stable high-contrast surface for the typography.

3. Responsive Scaling of Media

The Challenge: The "Hero" images and the background video often lost their focal points or became distorted when switching from a wide desktop monitor to a narrow mobile screen.

The Resolution: I utilized the object-fit: cover CSS property. This ensured that images and videos filled their assigned dimensions without stretching. Additionally, I used Flexbox (display: flex with flex-wrap: wrap) to allow gallery cards to flow naturally into a single column on smaller devices without needing complex math.

Lessons Learned

Semantic HTML is a Foundation, Not an Option

Overcoming accessibility hurdles taught me that using the right tags (like "article" for the report and "nav" for links) isn't just for organization it's the primary way to ensure the site is "perceivable" for screen reader users. I learned that building with accessibility in mind from the start is much easier than trying to "fix" it later.

The Power of "Dry" Code

(Don't Repeat Yourself). The transition from multiple modals to a single dynamic modal taught me the value of efficient scripting. It made the project much easier to debug and showed me how JavaScript can be used to keep HTML clean and manageable.

User Experience (UX) vs. Visual Flare

I learned that while high-end features like autoplaying videos look great, they must be implemented responsibly (using muted and playsinline). A design that interrupts a user’s experience or makes content unreadable is an unsuccessful design, no matter how "cool" it looks.

Compliance

The implementation of the Blue Horizon project was guided by a strict adherence to the Janet Acceptable Use Policy (AUP) and the University of Westminster’s digital governance standards. By focusing exclusively on the educational dissemination of UN Sustainable Development Goal 14 the site remains non-commercial and aligned with the network's mission for research and learning. To ensure legal and ethical compliance I prioritized intellectual property rights by providing clear attributions for all external media assets and maintained a high standard of accessibility through semantic HTML5. Ultimately the project demonstrates a professional commitment to network integrity and user inclusivity ensuring that the content is both secure and perceivable for a diverse academic audience.See the lecture notes for more details on compliance.
