Feedback

Technical Description

The Feedback Page is made with our team website template, so it looks the same as the other pages. The page has three main sections. The first section is a scrollable list that shows many programme cards. Flexbox is used to arrange these cards neatly.Programme cards were built using Div The second section is the reviews area. The review cards can move sideways because the page uses horizontal scrolling. The third section is the feedback form.JavaScript is used to make a live character counter. This counter changes the number while the user is typing. Buttons and cards have hover animations to make the page feel more interactive.I used input types text, email, tel, date, and datetime-local and grouped the content using fieldsets.The whole page uses structured HTML, simple CSS for layout and style, and JavaScript for small interactive features. This helps the page meet all the coursework requirements.

HTML Structure

  • Main tags - Header , Nav , Main , Section , Form , Footer
  • nav - Holds navigation links such as Home, Gallery, Team, Feedback, etc. This helps users easily move between pages.
  • section - Used to divide the page section (Cards,Reviews,Ratings,Feedback)
  • form - Used to collect user feedback.
  • fieldset and legend - To keep the form clear and organised
  • button - Used to submit the form,learn more information,See more reviews
  • select and option - Used to create a dropdown

CSS Structure
  • Used a linear gradient background to create an ocean theme design
  • Used border radius to create rounded corner
  • Used margin and padding for proper spacing between elements
  • Added hover styles to buttons and navigation links
  • Applied transition: all 0.3s ease for smooth animations
  • Designed scrollbar using ::-webkit-scrollbar

Accessibility

All images in programme cards include descriptive alt attributes, All inputs have label tags associated directly with the form fields to help assistive technologies and screen readers. The colours have good contrast, with light text on a dark background. The programme list and reviews can scroll, so the page is not too long. Buttons and cards are big and easy to tap on mobile

Team Page

Technical Description

The Team Page was designed using the shared website layout, including the fixed navigation bar and footer. Four team members are shown in cards, arranged using Flexbox so they line up neatly. Each team member's details are displayed like this,

  • Each member's profile image
  • Member's name
  • Member's role
  • A hidden details section that expands on hover
CSS hover effect is used to animate the cards
  • Increasing scale
  • Applying box-shadow
  • Slight upward movement

HTML Structure

  • Main tags - Header , Nav , Main , Section , Footer

CSS Structure
  • Used justify-content:center to align team members
  • Used margin and padding for perfect spacing
  • Displayed hidden details when hovering over a card

The page has a dark gradient background and soft, transparent cards to match the rest of the site. The design works on all screen sizes, with round images, clear titles, and even spacing

Accessibility

The Team Page follows several accessibility practices to ensure all users can navigate and understand the content. Each team member's image includes descriptive alt text.Cards are keyboard-accessible using tabindex="0", allowing users who cannot use a mouse to focus on each member.Large,clear headings (h1, h2) help screen readers understand the structure of the page.

Challenges and Lessons Learned

Challenges

  1. JavaScript Text Area Features
    Adding a live character count in a text area needs careful coding. It must update every time the user types or deletes text, and it should always show the correct number without breaking.
  2. Vertical Scrolling
    Handling long pages with scrollable content needs careful testing. The scrolling should feel smooth, and the layout must stay correct on different devices.
  3. Hover Effects
    Adding interactive effects when users hovered over buttons or links required precise CSS
  4. Margins & Padding
    Getting the spacing between elements right was challenging. Wrong margins or uneven padding can make the page look messy or unbalanced.

Learned

  1. CSS Decorations
    Learned how to use shadows, borders, blur effects, and backgrounds effectively to enhance visual appeal without compromising usability.
  2. Transitions
    Learned how to add smooth animations to improve user experience and make interactions feel natural.
  3. Navigation Design
    Learned to structure menus, links, and sections so users can easily find information.

Compliance

The Content Page was created following responsible and ethical web practices. All content is suitable, educational, and relevant to the project's theme of sustainable ocean practices. External resources and images were used carefully, with correct attribution and no copyright violations. The page shows professional behavior by presenting original work and properly citing information sources. Accessibility and usability were also considered, making the page inclusive for all users. This shows a commitment to producing web content that is safe, ethical, and trustworthy.
